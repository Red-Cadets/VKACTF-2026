## Cold Storage

| Событие | Название | Категория |
| :------ | :------- | :-------- |
| VKACTF 2026 | Cold Storage | pwn |

### Описание

> Автор: Invis_one
>
> После холодной перезагрузки системы мы не можем прочитать файл в корне системы...


### Решение

В запросе обновления пути передаются количество символов `PathChars` и сам путь. Драйвер проверяет размер так:

```c
bytes = (USHORT)((PathChars + 1) * sizeof(WCHAR));

if (bytes > sizeof(State.Scratch))
    return STATUS_NAME_TOO_LONG;
```

Если указать `PathChars = 0xffff`, результат вычисления переполняется и после приведения к `USHORT` становится равен нулю. Проверка проходит, но затем драйвер копирует фактический размер входного буфера в `Scratch`.

Сразу после `Scratch` в глобальной структуре находится поле `SourcePath`:

```c
typedef struct _VSTORCACHE_STATE {
    WCHAR Scratch[0x80];
    WCHAR SourcePath[0x100];
} VSTORCACHE_STATE;
```

Таким образом, переполнение `Scratch` позволяет заменить `SourcePath`. Первые байты нагрузки должны пройти проверку разрешенного префикса, а по смещению `0x100` можно записать новый корневой каталог:

```text
PathChars = 0xffff

offset 0x000: \??\C:\ProgramData\VStorCache\cache\ok
offset 0x100: \??\C:
```

После обновления пути операция `fetch` с именем `flag.txt` открывает уже `\??\C:\flag.txt`. Чтение выполняется драйвером от имени `SYSTEM`, поэтому ограничения доступа к файлу не мешают получить флаг.


Полный пример решения представлен в [solve.py](solve.py).
