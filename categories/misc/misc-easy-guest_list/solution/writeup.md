## Guest List

| Событие | Название | Категория |
| :------ | ---- | ---- |
| VKACTF 2026 | Guest List | misc/forensics |

### Описание

> Автор: Invis_one
>
> После полуночи в офисе аккредитации Олимпийских игр был распечатан список
> приглашенных гостей.

### Решение

В `browser_history.sqlite` находится страница предварительного просмотра с
параметром `job=42`. Этот же номер встречается в архивах журналов CUPS:

```bash
sqlite3 browser_history.sqlite "select url, title from urls;"
tar -xzf printer_logs.tar.gz
grep -R "42" .
```

Из журналов узнаем имя документа `olympic_guest_list_final.pdf`. Далее открываем
`olympic-print.pcap` в Wireshark и находим TCP-соединение с портом `631`, который
используется CUPS/IPP. Через **Follow TCP Stream** виден запрос печати и
gzip-сжатая часть документа. Начало gzip определяется по сигнатуре `1f 8b 08`.

Сохраняем данные от этой сигнатуры и распаковываем их. Получается начало PDF,
однако документ обрывается.

Оставшаяся часть находится в spool-архиве:

```bash
tar -xzf spool_recovered.tar.gz
```

Файл `c00042` подтверждает номер задания, а `d00042-001.tail` содержит конец
документа. Склеиваем обе части:

```bash
cat pdf_prefix.bin d00042-001.tail > recovered_guest_list.pdf
```

Внутри восстановленного PDF текст страницы хранится в потоке с фильтром
`FlateDecode`. После его распаковки среди PDF-команд находится невидимая белая
строка `audit marker`, содержащая индивидуальный флаг.

Готовый скрипт решения представлен в [solve.py](solve.py):

### Флаг

```text
vkactf{the_guest_list_was_never_deleted}
```
