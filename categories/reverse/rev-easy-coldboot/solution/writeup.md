## Coldboot

| Событие | Название | Категория |
| :------ | ---- | ---- |
| VKACTF 2026 | Coldboot | reverse |

### Описание

> Автор: Invis_one
>
> После холодной перезагрузки старое сетевое устройство перестало входить в диагностический режим. От него остался только небольшой фрагмент файловой системы.


### Решение

В раздаче находится фрагмент файловой системы устройства. Основной интерес представляет файл `rootfs/sbin/watchdogd`.

Команда `file` показывает, что это статически собранный `MIPS32 big-endian ELF`. Запустить его можно через QEMU:

```bash
qemu-mips ./rootfs/sbin/watchdogd
```

При обычном запуске получаем сообщение:

```text
watchdogd: diagnostic interface locked
```

Открываем бинарный файл в Ghidra или IDA и анализируем функцию `main`. По строкам и обращениям к файловой системе видно, что программа читает следующие файлы:

```text
etc/board.conf
etc/calib.dat
etc/nvram.bin
var/state/boot.slot
```

Основное состояние устройства хранится в `nvram.bin`. Его структура имеет следующий вид:

```c
struct nvram {
    char magic[4];
    uint16_t version;
    uint16_t profile;
    uint32_t board_crc;
    uint32_t boot_counter;
    uint8_t seed[16];
    uint8_t state[64];
    uint32_t checksum;
};
```

Все числовые поля записаны в big-endian. Последние 4 байта являются контрольной суммой:

```c
checksum = crc32(nvram_without_checksum) ^ 0x4d495053;
```

Поле `state` зашифровано XOR-потоком. Начальное состояние генератора вычисляется из серийного номера устройства, содержимого `calib.dat`, `seed`, аппаратной ревизии и активного boot slot. Далее для каждого байта вызывается `xorshift32`, а полученный байт XOR-ится с `state`.

После реализации этого алгоритма расшифрованное состояние начинается так:

```text
mode  = SAFE
level = 1
```

Чтобы включить диагностический режим, меняем поля на:

```text
mode  = DIAG
level = 0x1337
```

Остальные данные, в том числе `token_key` и проверочный тег, оставляем без изменений. После этого необходимо зашифровать `state` тем же потоком, записать его обратно в `nvram.bin` и пересчитать контрольную сумму файла.

Снова запускаем сервис:

```bash
qemu-mips ./rootfs/sbin/watchdogd
```

Теперь программа принимает состояние NVRAM и расшифровывает диагностический токен:

```text
watchdogd: diagnostic token: vkactf{...}
```

### Флаг

```text

```
