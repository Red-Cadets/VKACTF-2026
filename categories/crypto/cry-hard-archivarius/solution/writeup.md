## crypto-archivarius

| Событие | Название | Категория |
| :------ | ---- | ---- |
| VKACTF 2026 | Archivarius | crypto |

### Решение

Сервис принимает файл через multipart-загрузку, архивирует его (со сжатием) и возвращает размер результата в заголовке ответа. Флаг присутствует в данных, которые сжимаются вместе с нашим вводом.

Это классическая атака типа **CRIME/BREACH**: если сжатие применяется до шифрования (или размер сжатых данных доступен атакующему), можно восстановить секрет побайтово, эксплуатируя то, что совпадающие подстроки сжимаются лучше.

**Принцип атаки**

Алгоритм сжатия (LZ-семейство) заменяет повторяющиеся подстроки ссылками. Если наш ввод содержит правильный следующий байт флага, суммарный размер сжатых данных окажется меньше, чем при неверном байте — повторение будет длиннее.

**Алгоритм восстановления флага**

Известно начало флага: `vka{`. Для каждого следующего байта:

1. Перебираем все символы из алфавита (`0-9`, `a-z`, `A-Z`, `_`, `}`).
2. Для каждого символа отправляем на сервер: `padding + known_flag + candidate + "#"`.
3. Смотрим на размер архива в ответе.
4. Символ, давший **наименьший** размер, добавляется к флагу.

**Роль паддинга**

Чтобы сигнал был стабильным, к данным добавляется случайный паддинг из байт вне диапазона ASCII (`128–255`). Это вытесняет флаг в позицию, где компрессор точно использует совпадение с нашим вводом, а не с другими данными в потоке. Длина паддинга перебирается от 1 до 5000 — берётся та, при которой наблюдается чёткое разделение между лучшим и вторым кандидатом.

```python
if sorted_sizes[0][1] < sorted_sizes[1][1]:
    best_char = sorted_sizes[0][0]
    break
```

Процесс повторяется до тех пор, пока флаг не заканчивается на `}`.
