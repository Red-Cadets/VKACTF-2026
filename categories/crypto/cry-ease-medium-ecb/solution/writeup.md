## crypto-кбод

| Событие | Название | Категория |
| :------ | ---- | ---- |
| VKACTF 2026 | КБОД | crypto |

### Решение

Сервис позволяет создавать токены для email-адресов и проверять их. Токен — это ECB-зашифрованная строка вида:

```
email=<input>&uid=10&role=user
```

Так как режим ECB шифрует каждый 16-байтный блок независимо, можно провести атаку **ECB cut-and-paste**: собрать токен из блоков, полученных от разных запросов.

**Шаг 1. Получаем блок с `admin` + PKCS#7-паддингом**

Нужно, чтобы слово `admin` оказалось в начале отдельного блока шифротекста. Длина префикса `email=` — 6 байт, значит до границы блока нужно ещё 10 байт. Отправляем:

```
email = "A" * 10 + "admin" + \x0b * 11
```

Plaintext разбивается так:

```
[email=AAAAAAAAAA] [admin\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b\x0b] [...]
```

Второй блок шифротекста — это зашифрованный `admin` с валидным PKCS#7-паддингом.

**Шаг 2. Получаем токен, где `role=` начинается с границы блока**

Нужно, чтобы `role=user` начиналось ровно с начала нового блока. Длина `email=` + `&uid=10&role=` = 6 + 13 = 19 байт. Значит, input должен быть длиной 13 байт:

```
email = "B" * 13
```

Plaintext:

```
[email=BBBBBBBBBBBBB] [&uid=10&role=use] [r...]
```

Берём первые два блока шифротекста.

**Шаг 3. Склеиваем поддельный токен**

```python
forged_token = prefix_blocks + admin_block
```

После расшифровки сервер видит:

```
email=BBBBBBBBBBBBB&uid=10&role=admin
```

Проверяем токен и получаем флаг.
